# チームTODO セットアップ手順

博多ステラ歯科スタッフ向けの共有TODOリスト（Notion風カンバンボード）のセットアップ方法です。

## 構成

- **データ保存**: Google Sheets（1つのスプレッドシートに `Tasks` シートと `Comments` シートを自動作成）
- **バックエンド**: Google Apps Script（Webアプリとしてデプロイ、無料・サーバー不要）
- **フロントエンド**: 静的 HTML/JS/CSS（`web/` ディレクトリ）

## 1. Google スプレッドシートを作成

1. [Google ドライブ](https://drive.google.com/) で新規スプレッドシートを作成
2. 名前を「CLAUDENOTE チームTODO」などに変更
3. シート ID をメモ（URLの `/d/...../edit` の部分）

## 2. Apps Script を設定

1. 上記スプレッドシートを開いた状態で、メニューから `拡張機能 > Apps Script` を選択
2. 既存のコードを全削除し、リポジトリ内の `apps_script/Code.gs` の内容をコピペ
3. 左側の歯車（プロジェクトの設定）から「`appsscript.json` マニフェストファイルを表示する」にチェック
4. `appsscript.json` の内容を `apps_script/appsscript.json` の内容で置き換え
5. 一度保存（💾）し、関数 `ensureSheets_` を選択して「実行」、権限を承認
   - Tasks / Comments シートが自動生成されます

### スタッフ・タグの初期値（任意）

Apps Script の画面で `プロジェクトの設定 > スクリプト プロパティ` から以下を登録すると、Web UI の候補に出ます。

| プロパティ名 | 値の例 |
| --- | --- |
| `STAFF` | `院長, 田中, 山田, 佐藤` |
| `TAGS` | `受付, 診療, 経理, 緊急` |
| `STATUSES` | `未着手, 進行中, レビュー, 完了`（既定） |
| `PRIORITIES` | `高, 中, 低`（既定） |

## 3. Web アプリとしてデプロイ

1. Apps Script 画面右上の「デプロイ」→「新しいデプロイ」
2. 種類: 「ウェブアプリ」を選択
3. 設定:
   - 説明: `CLAUDENOTE TODO API`
   - 次のユーザーとして実行: **自分**
   - アクセスできるユーザー: **全員**（社内利用なら「同じドメインのユーザー」でも可）
4. デプロイをクリック → アクセス権限を承認
5. 表示された **ウェブアプリ URL** をコピー（`https://script.google.com/macros/s/.../exec`）

⚠️ コードを修正したら「デプロイの管理」から既存のデプロイを編集して「新しいバージョン」で再デプロイしてください。

## 4. Web アプリを開く

### A. ローカルで開く

`web/index.html` をブラウザで直接開いても動きます（CORS制限は Apps Script 側が対応済み）。

### B. GitHub Pages で配信（推奨）

1. リポジトリの `Settings > Pages` で `main` ブランチの `/web` フォルダを公開
2. 発行されたURL（例: `https://<org>.github.io/CLAUDENOTE/`）をスタッフに共有

### 初回起動

1. ページを開くと設定モーダルが表示されます
2. **Apps Script Web App URL** に手順3でコピーしたURLを貼り付け → 保存
3. 「あなたの名前」欄に自分の名前を入力（コメントや更新履歴に使われます）

## 使い方

- **新規タスク追加**: 右上「＋ 新規タスク」または各カラムの「＋ このステータスに追加」
- **編集**: カードをクリック → モーダルで編集 → 保存
- **進捗変更**: カードをドラッグして別カラムへ（自動でステータス更新 + 履歴コメント追加）
- **コメント**: タスク詳細モーダルの下部から投稿
- **フィルタ**: 上部の検索ボックス、担当者、タグで絞り込み
- **再読み込み**: 「⟳」ボタン（60秒ごとに自動更新もあり）

## トラブルシューティング

| 症状 | 対処 |
| --- | --- |
| `未接続` のまま | 設定（⚙）で URL を確認。末尾が `/exec` になっているか |
| `HTTP 401/403` | デプロイのアクセス権限を「全員」に変更、または同ドメインに統一 |
| コード修正が反映されない | デプロイの管理から「バージョン: 新しいバージョン」で再デプロイ |
| シートが見当たらない | Apps Script で `ensureSheets_` を一度手動実行 |

## データ構造（Sheets）

### Tasks シート

| 列 | 内容 |
| --- | --- |
| id | UUID（自動） |
| title | タイトル |
| necessity | やる必要性（背景・目的） |
| assignee | 担当者 |
| due | 期日（YYYY-MM-DD） |
| status | 未着手 / 進行中 / レビュー / 完了 |
| priority | 高 / 中 / 低 |
| tags | カンマ区切り（例: `受付, 緊急`） |
| createdAt | 作成日時 |
| updatedAt | 更新日時 |
| createdBy | 作成者名 |

### Comments シート

| 列 | 内容 |
| --- | --- |
| id | UUID（自動） |
| taskId | 対応するタスクID |
| author | 投稿者 |
| body | 本文（ステータス変更履歴も自動追加） |
| createdAt | 投稿日時 |

Sheetsを直接編集することも可能ですが、id 列は変更しないでください。

> 既に旧バージョンのスキーマでシートを作成していた場合は、Tasks シートの C 列（title の右隣）に空列を1つ挿入し、ヘッダーを `necessity` にしてください。Apps Script を再保存すれば追従します。
